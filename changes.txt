## 💻 3. Go Backend Codebases

### services/wallet/main.go
- **Resilient MongoDB Startup**: Made MongoDB Atlas connection and ping non-fatal during startup, enabling the wallet to boot in degraded mode and expose `/api/status` for observability even if MongoDB is offline or resolves incorrectly. Added `MONGO_URI` environment variable override support.
- **Asynchronous P2P Connection**: Converted the blockchain node connection process (`ConnectToBlockchain`) to run in a background goroutine, preventing startup blocking and allowing the web server to start instantly.
- **Auto-Discovery**: Dynamically resolves local P2P bootstrap node info from `BLOCKCHAIN_API_URL + "/api/p2p/info"` instead of hardcoded `localhost:8080`.
- **Status Connection Tracking**: Enhanced `/api/status` to call `wallet.DefaultBlockchainClient.IsConnected()` instead of returning a hardcoded `true` check, and optimized its database ping timeout from 5s to 1s to ensure snappy diagnostics responses and prevent connection drops.
- **Uptime Tracking**: Added global `startTime` variable to report accurate service runtime seconds in `/api/status`.

### bridge-sdk/main_bridge/main.go
- **Dynamic Helper Functions**:
  - Declared `getBlockchainURL()` as a package-level helper function to allow access from both `BridgeSDK` methods and `BlackHoleBlockchainInterface` methods.
  - Implemented `getWalletURL()` and `getBlockchainURL()` to retrieve URLs from environment variables (`WALLET_API_URL`, `BLOCKCHAIN_API_URL`) or fallback container host names in `DOCKER_MODE`.
- **Endpoint Routing & Proxy Updates**:
  - Aligned wallet connection checks in `handleWalletHealth`, `handleWalletDashboardStatus`, and `handleProxyWalletDashboardHealth` to query the correct `/api/status` path instead of non-existent `/api/health`.
  - Refactored `handleProxyWalletDashboardWallets` and `handleProxyWalletDashboardTransactions` to call `/api/wallets` and `/api/transactions` through the dynamic helper.
  - Aligned all occurrences of hardcoded `"http://localhost:8080"` with `getBlockchainURL()` across handlers (including `handleMainDashboardStatus`, `handleProxyMainDashboardHealth`, `handleProxyMainDashboardBlockchain`, `handleProxyMainDashboardNode`, `handleProxyMainDashboardWallets`, `handleProxyMainDashboardActivities`, `handleWalletTransactions`, `processTransactionViaHTTP`, `getStatsViaHTTP`, `verifyTargetChainTransaction`, and `checkBlackholeConnection`), enabling container-to-container connectivity.
- **Out-of-Bounds Panic Fix**: Added safety length check when slicing wallet address keys in `handleProxyMainDashboardActivities` to prevent `slice bounds out of range` panics when processing short address strings (such as custom wallet names).
