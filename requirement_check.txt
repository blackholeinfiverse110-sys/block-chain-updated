Day 1 (combines Day 1+2) — Schemas, Endpoints,
Idempotency, Basic E2E
Goal: Stand up the Bridge service with stable schemas, HTTP endpoints, in-process mock listeners,
and idempotent relay into BlackHole.
Build steps
1. API schemas
◦ Define api-schema.proto (v1alpha1): BridgeMessage, SignedBridgeMessage,
RelayResponse, StatusResponse.
◦ Generate stubs (gRPC + JSON/REST gateway) into bridge/api/gen/.
2. HTTP/REST endpoints (backed by the same handlers used by gRPC):
◦ POST /relay/eth, POST /relay/sol — accept SignedBridgeMessage.
◦ GET /bridge/status — health, height, queueDepth.
◦ GET /log/retry — last N retry items.
◦ GET /log/events — recent accepted event summaries.
3. Idempotency + replay protection
◦ bridge/core/dedupe: BoltDB (or Badger) keyed by eventHash; reject duplicates.
◦ bridge/core/retry: exponential backoff, max attempts, dead-letter queue file bridge/
retry-dlq.jsonl.
4. Minimal in-process listeners (mock)
◦ bridge/core/listener/eth_mock.go, sol_mock.go produce Transfer-like events
(configurable rate).
◦ Pipe into Relay() and persist acceptance to bridge/internal/storage.
5. Structured logs + metrics
◦ Zap logger with fields: chain, txHash, eventHash, latency_ms, attempt.
◦ Prometheus-style counters in memory; write snapshot to deploy/bridge/metrics.json
every 30s.

◦ POST /relay/eth accepts a sample message and logs accepted.
• Files required: api-schema.proto, handlers, dedupe, retry, mock listeners, bridge-api.md

Day 2 (combines Day 3+4) — Cross-Chain Proof, Latency,
Failure Injection, Docker
Goal: Prove ETH→BH→SOL and reverse with measurable latency, robust retries, failure injection,
and one-command Docker run.
Build steps
1. Cross-chain simulation
◦ Script scripts/simulate.sh to emit ETH-like events, relay to BH, mirror to SOL
listener.
◦ Persist “eventRoot” per 10 events (Merkle root or rolling hash) to bridge/internal/
roots/.
2. Latency + SLO
◦ Per event compute t_ingest, t_commit, latency_ms. Export rolling p50/p90/p99 to
metrics.json.
◦ CLI: bridgectl stats prints SLOs from metrics.json.
3. Failure injection
◦ Config flags: --inject-drop=0.03, --inject-delay-ms=50-200.
◦ Ensure retries converge; DLQ remains near zero under 10k events.
4. Dockerize
◦ deploy/bridge/Dockerfile multi-stage build.
◦ deploy/bridge/docker-compose.yml with service bridge, volume for BoltDB, port
mappings.
◦ .env parameters: RPC urls (mock), retry limits, batch size, failure-inject knobs.
5. Docs
◦ Expand bridge-api.md: endpoint examples (curl), error codes, dedupe policy, retry
semantics, SLO targets.

Pre-Prod Readiness

◦ Implement Ed25519 verification; reject unsigned or invalid messages.

◦ Graceful shutdown with draining queues and persisted offset checkpoints.
4. Release packaging
◦ make bridge target; make docker image tag blackhole/bridge:v0.3-rc1.
◦ bridge/CHANGELOG.md with upgrade notes.
